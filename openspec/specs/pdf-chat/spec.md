# Capability: pdf-chat

## Purpose

Provides a RAG-based conversational interface for asking questions about a specific paper's PDF content, with markdown and LaTeX rendering in responses.

## Requirements

### Requirement: Chat panel layout

The chat panel SHALL appear to the right of the PDF viewer in a horizontal split-pane layout when chat mode is active. The PDF viewer SHALL occupy the left portion and the chat panel SHALL occupy the right portion. The split SHALL be resizable by dragging the divider.

#### Scenario: Enter chat mode

- **WHEN** user activates chat mode via the toolbar toggle
- **THEN** the detail view switches to a horizontal split with PDF left and chat panel right

#### Scenario: Exit chat mode

- **WHEN** user deactivates chat mode via the toolbar toggle
- **THEN** the view returns to the standard detail layout (metadata + PDF vertical stack)

#### Scenario: Resize split

- **WHEN** user drags the split divider
- **THEN** the PDF and chat panel resize proportionally

### Requirement: Greeting and summary

When chat mode is opened for a paper, the chat panel SHALL display a greeting area at the top containing the paper title and a bullet-point summary derived from the paper's existing AI summary. Below the summary, a "Summarize this paper" button SHALL be displayed. Clicking it SHALL send a summarization request to the AI provider.

#### Scenario: Paper with existing summary

- **WHEN** chat mode opens for a paper that has an AI summary
- **THEN** the greeting shows the paper title and summary as bullet points

#### Scenario: Paper without summary

- **WHEN** chat mode opens for a paper with no AI summary
- **THEN** the greeting shows the paper title and a "Summarize this paper" button without bullet points

#### Scenario: Summarize button clicked

- **WHEN** user clicks "Summarize this paper"
- **THEN** a summarization request is sent to the AI provider and the response appears as a chat message

### Requirement: Suggested questions

After document indexing completes, the chat panel SHALL display 2-3 suggested questions as clickable cards below the greeting area. The questions SHALL be generated by the AI provider at index time and stored with the document index. Clicking a suggested question SHALL send it as a user message.

#### Scenario: Suggested questions displayed

- **WHEN** indexing completes for a paper
- **THEN** 2-3 suggested question cards appear below the greeting

#### Scenario: Click suggested question

- **WHEN** user clicks a suggested question card
- **THEN** the question is sent as a user message and the cards disappear

#### Scenario: Indexing not yet complete

- **WHEN** chat mode opens and the paper has not been indexed yet
- **THEN** suggested questions are not shown until indexing finishes

### Requirement: Message display

The chat panel SHALL display a scrollable list of messages. User messages SHALL appear right-aligned. Assistant messages SHALL appear left-aligned with an assistant avatar. The message list SHALL auto-scroll to the latest message when a new message arrives.

#### Scenario: User sends a message

- **WHEN** user sends a question via the input bar
- **THEN** the message appears right-aligned in the conversation

#### Scenario: Assistant responds

- **WHEN** the AI provider returns a response
- **THEN** the response appears left-aligned with an assistant avatar

#### Scenario: Auto-scroll

- **WHEN** a new message is added to the conversation
- **THEN** the message list scrolls to show the latest message

### Requirement: Citation rendering

Assistant messages SHALL parse page references in the format `[p.N]` (and variants like `[p. N]`, `[page N]`, `[pp. N-M]`) and render them as clickable teal citation badges. Each badge SHALL display the page number(s).

#### Scenario: Single page citation

- **WHEN** an assistant message contains `[p.5]`
- **THEN** it renders as a clickable teal badge showing "p.5"

#### Scenario: Page range citation

- **WHEN** an assistant message contains `[pp. 3-5]`
- **THEN** it renders as a clickable teal badge showing "pp. 3-5"

#### Scenario: No citations

- **WHEN** an assistant message contains no page references
- **THEN** the message renders as plain text without citation badges

### Requirement: Citation navigation

Clicking a citation badge in a chat message SHALL scroll the PDF viewer to the referenced page.

#### Scenario: Navigate to cited page

- **WHEN** user clicks a citation badge `[p.5]`
- **THEN** the PDF viewer on the left scrolls to page 5

#### Scenario: Navigate to page range

- **WHEN** user clicks a citation badge `[pp. 3-5]`
- **THEN** the PDF viewer scrolls to page 3

### Requirement: Message input

The chat panel SHALL include a text input bar at the bottom with a text field and a send button. Pressing Enter or clicking the send button SHALL submit the message. The input SHALL be disabled while the AI is generating a response, showing a loading indicator.

#### Scenario: Send message

- **WHEN** user types a question and presses Enter
- **THEN** the message is sent and the input field clears

#### Scenario: Loading state

- **WHEN** the AI provider is generating a response
- **THEN** the input field is disabled and a loading indicator is shown

#### Scenario: Empty input

- **WHEN** user presses Enter with an empty input field
- **THEN** nothing happens (no message sent)

### Requirement: Conversation persistence

Chat conversations SHALL be persisted per paper. When reopening chat mode for a paper, the previous conversation SHALL be restored. Users SHALL be able to start a new conversation, which archives the previous one.

#### Scenario: Reopen chat

- **WHEN** user opens chat mode for a paper with a previous conversation
- **THEN** the previous conversation messages are displayed

#### Scenario: Start new conversation

- **WHEN** user clicks "New conversation"
- **THEN** a new empty conversation starts and the previous one is archived

### Requirement: RAG query pipeline

When the user sends a message, the system SHALL embed the query, retrieve the top-k most relevant chunks (k=5-8) from the paper's index, and send them as context alongside the user's question to the AI provider. The system prompt SHALL instruct the model to cite page numbers using `[p.N]` format.

#### Scenario: Query with indexed paper

- **WHEN** user sends a question about an indexed paper
- **THEN** relevant chunks are retrieved and included as context in the AI request

#### Scenario: Query without index (fallback)

- **WHEN** user sends a question but the paper has no embedding index (provider doesn't support embeddings)
- **THEN** the full extracted PDF text is included as context instead (context-stuffing fallback)
